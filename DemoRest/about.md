# REST and CQRS pattern 

## REST

REST API - rules for building and interacting with web services. 

Client-server. Representational State Transfer, uses HTTP HTTPS.

- Stateless (state is not shared between requests - scalable)

- Client-server architecture.

- Cacheble. 

## CQRS pattern -> NEXT TIME

CQRS - Command and queries responsibility separation.

Api complexity -> not well scaling. 

CQRS - read operations (query) and write operations (commands) are entirely separated. 

Web development pattern. Separation of concerns.  

Scalability. More complex patterns/better design/.
